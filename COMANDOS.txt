# COMANDOS PARA EXECUTAR E TESTAR

## INICIAR INFRAESTRUTURA

```bash
docker-compose up -d
```

Aguarde 10 segundos para tudo estar pronto:
```bash
sleep 10
```

## 1. TESTES UNITÁRIOS DO MIDDLEWARE

```bash
cd middleware
go test -v
```

## 2. TESTES DE CARGA - Go

### Compilar
```bash
go build -o load_test load_test_main.go
```

### Executar - Cenário LIGHT (10 usuários)
```bash
./load_test -scenario=light
```

### Executar - Cenário MEDIUM (50 usuários)
```bash
./load_test -scenario=medium
```

### Executar - Cenário HEAVY (100 usuários)
```bash
./load_test -scenario=heavy
```

### Executar - Cenário STRESS (500 usuários)
```bash
./load_test -scenario=stress
```

### Executar - TODOS OS CENÁRIOS
```bash
./load_test -scenario=all
```

### Customizado (exemplo: 75 usuários, 1s think time)
```bash
./load_test -users=75 -requests=100 -think=1s
```

## 3. TESTES EM Go (Benchmarks)

```bash
go test -bench=BenchmarkLoadTest -benchtime=1x ./handlers
```

## 4. PARAR INFRAESTRUTURA

```bash
docker-compose down
```

## ESTRUTURA DE ARQUIVOS CRIADOS

- `load_test_main.go` - Executável de testes de carga (100% Go)
- `middleware/main_test.go` - Testes unitários do middleware
- `handlers/load_test.go` - Testes de carga (benchmarks)
- `middleware/dockerfile` - Container do middleware (já estava)
